多线程

使用多线程

可以把线程看做成一个箭头，每多一个线程，就多出一个在运行代码的箭头

守护线程

如果某个子线程的daemon属性为False，主线程结束时会检测该子线程是否结束，如果该子线程还在运行，则主线程会等待它完成后再退出；默认就是False

如果某个子线程的daemon属性为True，主线程运行结束时不对这个子线程进行检查而直接退出，同时所有daemon值为True的子线程将随主线程一起结束，而不论是否运行完成。

线程阻塞

如果一个线程在执行过程中要执行另外一个线程，并且等其完成以后才能接着执行

线程锁

多个线程在共享一个全局变量，同时对全局变量进行修改时，会发生数据错误，要使用线程锁，当一个线程在使用变量时其他线程不能使用

迭代器

创建一个迭代器

运行结果

首先for循环会调用isinstance(classmate,Iterable)判断该对象是否是一个可迭代对象，如果是可迭代对象，再调用iter(classmatte)获取该对象对应的”迭代器”。用isinstance(获取的"迭代器",iterator)判断该对象对应的”迭代器”是否是一个真正的迭代器，最后判断是一个真正的迭代器，再调用next方法取值。

迭代器的应用

迭代器的用处是为了节省空间，在迭代对象中存贮着生成数据的方法，每次需要数据的时候去迭代器中取就行了，只占用生成数据的所需的内存空间。如果是列表数据越大则占用空间越大；

运行结果

100个Fibonacci数列已经很大了，如果是1000个全都用列表存起来，要占用巨大的内存空间，而迭代器只是占用了生成数据方法的内存空间

生成器

最简单的生成器

这是一个生成器，同时也是一个特殊的迭代器，它是可以被迭代的

生成器的应用

运行结果

看上去是一个普通的函数，但是当一个函数有了yield之后，他就不是一个函数了，而是一个生成器。函数运行时，从上往下执行，当执行到yield时会立刻返回a的值给i，然后进行下一次for循环，这一次函数从yield下面开始执行也就是a, b = b, a + b开始，当再次执行到yield时会立刻返回a的值给i，以此往复。函数执行结束，for循环结束

yiled实现多任务

运行结果

首先主程序在循环中用next调用test1生成器，遇到yield返回，接着主程序的循环用next调用test2生成器，遇到yield返回，主程序的循环继续用next调用test1生成器，这样就实现了多任务

gevent实现多任务

gevent比yield更加智能

运行结果

首先要用gevent.monkey模块给程序打补丁，让gevent能捕捉到程序中所有需要等待的地方。gevent如果遇到需要等待的地方会立马切换，这也就实现了多任务
